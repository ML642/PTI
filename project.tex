\section{Wstępne pomiary}

\subsection{Pozyskiwanie danych}
Podobnie jak w projekcie o bilardzie kluczowe było pozyskanie zdjęć, tutaj kluczowe jest pozyskanie logów pogodowych. Dane pobieramy z:
\begin{itemize}
    \item \textbf{Open-Meteo API:} Modele GFS i ECMWF (dane globalne).
    \item \textbf{Meteo.pl (ICM UW):} Model UM (dane wysokiej rozdzielczości dla Polski).
    \item \textbf{Dane SYNOP:} Rzeczywiste pomiary ze stacji meteorologicznych jako „Ground Truth”.
\end{itemize}

\subsection{Problemy pomiaru}
W trakcie zbierania danych zidentyfikowano problemy analogiczne do zakłóceń obrazu:
\begin{itemize}
    \item \textbf{Aliasing czasowy:} Różnice w godzinach publikacji „runów” modeli (00Z, 06Z, 12Z, 18Z).
    \item \textbf{Szum lokalizacyjny:} Różnica między punktem siatki modelu a fizycznym położeniem stacji (np. wpływ wysokości n.p.m.).
    \item \textbf{Błędy normalizacji:} Konieczność ujednolicenia jednostek (np. m/s na km/h).
\end{itemize}
